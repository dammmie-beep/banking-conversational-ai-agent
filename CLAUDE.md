# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the Application

```bash
# Install dependencies
pip install -r requirements.txt

# Initialize database from Excel (one-time setup)
python sqlite.py

# Start the Flask server
python run.py
```

Server runs on `http://0.0.0.0:5000` with threading enabled.

## Required Files (not in git)

Before running, ensure these large files are present:
- `models/qwen2.5-3b-instruct-q4_k_m.gguf` — Qwen2.5 3B quantized model (download from Hugging Face)
- `models/minilm/` — Cached `sentence-transformers/all-MiniLM-L6-v2` embedding model
- `data/banking.db` — SQLite database (generated by `python sqlite.py` from the Excel file)
- `data/Product_Information.txt` — Plain text product catalog

## API Endpoints

```
GET  /api/health                        — Health check
POST /api/chat                          — Main chat (body: {session_id?, message})
GET  /api/session/{session_id}/history  — Retrieve conversation history
DEL  /api/session/{session_id}/clear    — Clear session
```

## Architecture

```
POST /api/chat
  → routes.py          — HTTP validation, session_id generation
  → agent.run_agent()  — Core orchestration
      → route_with_llm()     — LLM classifies intent (pure LLM, no keyword fallback)
      → handle_block_card()  — Multi-turn card blocking flow
      → handle_account_query() — Account/transaction lookup with LLM summary
      → handle_product_query() — FAISS semantic search (RAG) + LLM answer
      → handle_general()     — Free-form conversational response
  → memory.py          — Per-session deque (10-turn sliding window)
  → tools.py           — Data access layer (SQLite queries + card blocking)
```

**Key design decisions:**
- `app/intent_router.py` exists but is **not used** — all routing is done purely by LLM in `agent.py`
- Sessions are **in-memory only** — lost on server restart
- LLM loads once as a singleton in `app/llm.py`; FAISS index built once at import in `agent.py`
- All tool functions return JSON dicts with `{"success": bool, ...}` or `{"error": "..."}` shape
- Card blocking is a multi-step flow tracked via `memory.set_state("current_flow", "block_card")`

## Configuration

All tunable parameters are in `config.py`:
- `MODEL_PATH`, `DB_PATH`, `PRODUCTS_PATH`, `EMBEDDING_MODEL` — file paths
- `N_CTX=8192`, `N_THREADS=4`, `MAX_TOKENS=200`, `TEMPERATURE=0.7` — LLM inference
- `WINDOW_SIZE=10` — conversation turns retained in memory

## LLM Behavior Notes

Qwen2.5-3B-Instruct reliably returns valid JSON for routing prompts. `agent.py` still includes:
- `_extract_first_json_object()` — strips extra text around JSON (safety net)
- `_looks_like_refusal()` — detects refusal patterns to trigger deterministic fallback for account queries

Response latency is 20–60 seconds on CPU.
